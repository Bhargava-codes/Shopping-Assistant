How to work this exercise

Start by explaining the flow

Trace a shopper request from the query, through the agent loop and local tools, to the final cart product and benchmark result.

Use failures as evidence

Pick two or three failed cases. For each one, identify what the shopper asked for, what the assistant output, what product was selected, and why that product failed.

Investigate retrieval

Check whether search returns useful candidates for the shopper's language. Think about ranking, category matching, synonyms, fuzzy matching, and whether broad token matches create distracting results.

Investigate tool contracts

Do not assume every tool response has the units, fields, or filtering behavior you expect. Compare tool output against the catalogue source when something looks suspicious.

Investigate verification

Look at when the agent verifies category, price, rating, stock, features, specs, and review evidence before adding a product to the cart.

Investigate selection

Decide how the assistant should compare multiple candidates, handle soft needs like quietness or comfort, and avoid claiming unsupported features.

Investigate cart safety

Check whether the cart action itself protects shoppers from invalid choices, or whether the agent must prove the product is valid before calling it.

Make a small change

Choose one focused improvement, explain the trade-off, rerun the benchmark, and update SUBMISSION.md with what changed and what still is not solved.
